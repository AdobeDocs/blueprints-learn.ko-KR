# 사용 사례 아이콘 생성기

블루프린트 저장소의 사용 사례 아이콘에 대한 자세한 아이콘 사양 및 AI 이미지 생성 프롬프트를 생성합니다.

## 이 스킬 사용 시기

- 아이콘이 필요한 새 사용 사례 만들기
- 아이콘 사양을 생성하기 위해 업계 사례 빌더 스킬에서 호출됩니다.
- 사용자가 사용 사례 아이콘 만들기, 업데이트 또는 교체에 대해 묻습니다.
- 사용자가 새 업계 카테고리에 대한 아이콘 배치를 생성하려고 합니다.

## 1단계: 입력 수집

사용자 또는 호출 스킬에서 다음 정보를 수집합니다.

**필수:**
- **사용 사례 이름** — 사람이 인식할 수 있는 사용 사례의 이름(예: &quot;포기한 장바구니&quot;, &quot;잠재 고객 점수&quot;)
- **업계 카테고리** — 자동차, b2b, 금융 서비스, 의료, 보험, 미디어 엔터테인먼트, 소매, 통신, 여행 접대 중 하나
- **간단한 설명** — 사용 사례를 설명하는 1-2개 문장

**선택 사항:**
- **선호하는 시각적 메타포 또는 주제** - 아이콘에 표시할 내용에 대한 특정 아이디어가 사용자에게 있는 경우

필수 입력이 누락된 경우 계속 진행하기 전에 사용자에게 문의하십시오.

## 2단계: 스타일 안내서 읽기

다음을 포함한 전체 스타일 가이드를 로드하려면 `references/icon-style-guide.md` 파일(이 스킬의 디렉터리에 대해 상대적)을 읽으십시오.

- 기술 사양
- 비주얼 스타일 지침
- 산업별 시각적 은유 예
- 프롬프트 템플릿
- 기존 아이콘 인벤토리

기존 아이콘과의 일관성을 유지하려면 스타일 안내서를 사용하십시오.

## 3단계: 아이콘 사양 생성

아이콘 사양의 세 부분을 모두 생성합니다.

### A. 파일 사양

사용 사례 이름 및 산업에서 파일 이름 및 경로를 가져옵니다.

- **파일 이름:** `icon-{kebab-case-name}.png`
   - 사용 사례 이름을 kebab-case(소문자, 공백은 하이픈)로 변환합니다.
   - 예: &quot;포기한 장바구니&quot;는 `icon-abandoned-cart.png`, &quot;교차 판매 상향 판매&quot;는 `icon-cross-sell-upsell.png`
- **경로:** `/help/blueprints/industry-use-cases/assets/use-case-icons/{industry}/`
- **차원:** 1024 x 1024픽셀
- **형식:** PNG, 8비트 RGB 또는 RGBA
- **대상 파일 크기:** 900KB - 1.4MB

### B. 이미지 생성 프롬프트

AI 이미지 생성 도구에 대한 자세한 복사 준비 프롬프트를 만듭니다. 프롬프트는 다음을 수행해야 합니다.

1. **명확한 시각적 메타포를 설명** — 사용 사례 개념을 즉시 전달하는 강력한 단일 시각적 메타포를 선택합니다. 사용자가 선호하는 은유를 제공한 경우 이를 사용하십시오. 그렇지 않으면 스타일 가이드 예제와 사용 사례 설명에 따라 하나를 선택하십시오.

2. **아티스틱 스타일을 지정** — 다음 스타일 지시문을 포함합니다.
   - 깨끗하고 현대적인 아이콘 일러스트레이션 스타일
   - 전문 기업 디자인 미학
   - 굵은 도형 및 깔끔한 선
   - 광택을 낸 완성가공품(비현실적)
   - 통합 설계 시스템과 일치

3. **기술 사양 포함:**
   - 정사각형 형식, 1024x1024 픽셀
   - 단일 중심 주제
   - 단색 또는 미묘한 그라데이션 배경
   - 주제와 배경 간의 높은 대비

4. **작은 크기 가독성 적용:**
   - 텍스트나 레터(어떤 종류든) 없음
   - 세밀한 세부 정보 또는 복잡한 패턴 없음
   - 복잡한 배경이나 다중 요소 장면 없음
   - 40px에서 선명하게 읽히는 볼드체 실루엣
   - 색상이 없어도 강력한 모양 인식

5. **색상표 안내:**
   - 전문적인 기업 환경에 적합한 색상
   - 활기차지만 세련됨(네온 또는 과하게 포화되지 않음)
   - 접근성을 위한 높은 대비
   - 동일한 업계 업종에서 다른 아이콘과의 결합

프롬프트를 이미지 생성 도구에 붙여넣을 수 있는 단일 단락으로 구성합니다.

### C. 카탈로그 참조 코드 조각

카탈로그 테이블에서 사용할 HTML 코드 조각을 생성합니다.

```
<img src="assets/use-case-icons/{industry}/icon-{name}.png" alt="{Alt Text}" width="40">
```

여기서 `{Alt Text}`은(는) 사람이 읽을 수 있는 사용 사례 이름입니다.

## 4단계: 동일 업계의 기존 아이콘 나열

스타일 안내서의 기존 아이콘 인벤토리 섹션을 확인하고 동일한 업계에 대한 현재 아이콘을 모두 나열합니다. 이렇게 하면 사용자가 다음과 같은 작업을 수행할 수 있습니다.
- 새 아이콘을 생성할 때 시각적 일관성 보장
- 이미 존재하는 아이콘을 복제하지 마십시오.
- 기존 아이콘을 스타일 예제로 참조

## 5단계: 출력 제공

다음 섹션으로 출력의 형식을 명확하게 지정합니다.

---

### 아이콘 사양: {Use Case Name}

**업종:** {Industry}

**파일 세부 정보:**
- 파일 이름: `icon-{kebab-case-name}.png`
- 경로: `/help/blueprints/industry-use-cases/assets/use-case-icons/{industry}/`
- 크기: 1024 x 1024픽셀
- 형식: PNG(8비트 RGB/RGBA)

**이미지 생성 프롬프트:**

> {복사하기 쉽도록 블록 따옴표로 형식이 지정된 완전한 세부 프롬프트}

**카탈로그 HTML:**

```html
<img src="assets/use-case-icons/{industry}/icon-{name}.png" alt="{Use Case Name}" width="40">
```

{Industry}의 **기존 아이콘:**
- {list of existing icon names}

**다음 단계:**
1. 위의 이미지 생성 프롬프트를 복사합니다.
2. 원하는 이미지 생성 도구(Adobe Firefly, Midjourney, DALL-E 또는 이와 유사한 )에 붙여 넣습니다.
3. 이미지를 생성하고 최상의 결과를 선택하십시오.
4. 필요한 경우 정확히 1024x1024 픽셀로 크기 조정/내보냅니다.
5. 위에 표시된 경로에 `{filename}`(으)로 저장합니다.
6. 40px 디스플레이 크기에서 아이콘이 선명하고 인식할 수 있는지 확인합니다.
7. 카탈로그 테이블에 이 사용 사례를 추가할 때 카탈로그 HTML 코드 조각을 사용하십시오.

## 가이드라인

- **실제 이미지를 생성하지 않음** — 이 스킬은 사양과 프롬프트만 생성합니다. 사용자는 외부 이미지 생성 도구를 사용해야 합니다.
- **사용 사례당 하나의 아이콘** — 각 사용 사례는 정확히 하나의 아이콘을 가져옵니다.
- **중복 항목을 확인** — 이름이 같거나 매우 비슷한 아이콘이 인벤토리에 이미 있으면 사용자에게 경고합니다.
- **가독성에 우선 순위 지정** — 자세한 메타포와 간단한 메타포 중 확실하지 않은 경우 항상 40px에서 잘 읽히는 간단한 옵션을 선택하십시오.
- **프롬프트에 고유해야 함** — 모호한 프롬프트는 일관되지 않은 결과를 생성합니다. 구체적인 시각적 세부 사항을 포함합니다(예: &quot;쇼핑 개념&quot;이 아닌 &quot;내부에 두 개의 다채로운 상자가 있는 장바구니&quot;).
- **가능한 경우 진부한 표현을 피하십시오** - 새롭지만 바로 인식할 수 있는 시각적 은유를 찾으십시오. 모든 &quot;스마트&quot; 사용 사례의 전구는 반복됩니다.
