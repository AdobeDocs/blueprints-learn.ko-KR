# 사용 사례 성숙도 평가 기준

## 완성도 수준

### 기본

**배지:** `[!BADGE Foundational]{type=Neutral}`

사용 사례는 다음과 같은 경우 **기본**&#x200B;입니다.

- 단일 단계 또는 이벤트 트리거 패턴(이벤트 트리거 메시징, 배치 아웃바운드 메시지 활성화)에 매핑
- 간단한 데이터 요구 사항(단일 이벤트 유형, 표준 프로필 속성)이 있음
- AI/ML 기능 최소 또는 불필요
- 복잡한 오케스트레이션 없이 표준 채널 게재(이메일, SMS, 푸시) 사용
- 은 명확한 모범 사례를 통해 잘 구축된 구현 패턴을 가지고 있습니다.
- 적은 시스템 통합 필요(일반적으로 1~2개의 데이터 소스)

**일반적인 패턴:** 이벤트 트리거 메시지, 일괄 아웃바운드 메시지 활성화

**예:** 포기한 장바구니 복구, 약속 알림, 서비스 알림, 가격 하락 알림, 청구서 결제 알림

### 출현

**배지:** `[!BADGE Emerging]{type=Informative}`

사용 사례는 다음과 같은 경우 **Emerging**&#x200B;입니다.

- 다단계 또는 개인화 패턴에 매핑 (다단계 오케스트레이션된 여정, 알려진 방문자 Personalization, 행동 권장 사항, 익명 방문자 Personalization)
- 행동 데이터 수집 및 분석 필요
- 추천 모델 또는 알려진 방문자 프로필 확인 사용
- 분기 논리를 사용하는 다중 터치 육성 시퀀스를 포함합니다.
- 중간 정도의 통합 복잡성(2~4개의 데이터 소스)
- 대상 활성화 또는 세그먼트 기반 타깃팅이 필요할 수 있음

**일반적인 패턴:** 다단계 오케스트레이션된 여정, 알려진 방문자 웹/앱 Personalization, 행동 권장 사항, 익명 방문자 웹 Personalization, B2B Audience Activation

**예:** 환영 시리즈, 약물 준수 캠페인, 개인화된 계정 대시보드, 잠재 고객 점수, 콘텐츠 권장 사항

### 고급

**배지:** `[!BADGE Advanced]{type=Caution}`

사용 사례는 다음과 같은 경우 **고급**&#x200B;입니다.

- 의사 결정 또는 크로스 채널 오케스트레이션 패턴에 매핑 (Decisioning을 사용한 크로스 채널 여정, Offer Decisioning)
- AI 기반 예측 또는 성향 점수 필요
- 여러 채널에서 동시에 중앙 집중식 의사 결정 논리 사용
- 여러 여정 지점에서 실시간 의사 결정을 통해 복잡한 오케스트레이션 수행
- 높은 통합 복잡성(4개 이상의 데이터 소스, 실시간 데이터 피드)
- 정교한 비즈니스 규칙, 우선 순위 관리 및 빈번한 거버넌스 필요

**일반적인 패턴:** Decisioning을 사용한 크로스 채널 여정, Offer Decisioning

**예:** AI 점수 책정, 수명 단계 제품 권장 사항, 교차 판매 최적화, 충성도 프로그램 개인화, VIP 독점 오퍼를 통한 이탈 방지

## 평가 워크플로

1. 사용 사례가 매핑되는 사용 사례 패턴 식별
2. 위의 &quot;일반적인 패턴&quot; 목록에서 빠른 일치 확인
3. 패턴이 성숙도를 명확하게 나타내지 않는 경우 데이터 복잡성, AI/ML 요구 사항 및 통합 범위를 평가합니다
4. &quot;패턴 매핑({pattern}) 및 {key factors}을(를) 기반으로 {reasoning}(으)로 인해 {level}(으)로 분류하겠습니다.&quot;
5. 계속하기 전에 사용자가 확인하거나 무시하도록 허용
