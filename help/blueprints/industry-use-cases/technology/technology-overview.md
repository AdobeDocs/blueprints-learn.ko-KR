# 기술 사용 사례

기술 조직은 Adobe Experience Platform을 사용하여 웹, 모바일 및 제품 표면의 데이터 수집을 중앙 집중화하고 실시간 이벤트를 제품 및 마케팅 프로그램을 지원하는 분석, 데이터 웨어하우스 및 활성화 대상으로 배포합니다. 기술 팀은 에지에서 이벤트 수집을 통합함으로써 클라이언트측의 복잡성을 줄이고 데이터 품질을 향상시키며 모든 다운스트림 시스템이 신뢰할 수 있는 단일 소스에서 일관된 동작 데이터를 수신하도록 합니다.

## 실시간 이벤트 전달

데이터 보강 및 활성화를 위해 Edge Network을 통해 수집된 실시간 행동 이벤트를 서드파티 분석, 데이터 웨어하우스 및 파트너 플랫폼으로 전달합니다. 에지에서 이벤트 수집을 중앙 집중화하고 여러 대상으로 전달하면 클라이언트측 태그 오버헤드가 줄어들고 데이터 품질이 향상되며 모든 다운스트림 시스템이 단일 권한 소스에서 일관된 이벤트 데이터를 수신하도록 합니다.

### 비즈니스에 대한 영향

실시간 이벤트 전달을 구현하는 기술 조직은 클라이언트측 태그 로드와 관련 성능 오버헤드를 줄이는 동시에 분석, Data Warehouse 및 활성화 대상 전반에서 이벤트 데이터의 일관성을 개선합니다. 또한 서버측 전달은 페이지 가중치를 줄이고 로드 성능을 향상시켜 사용자 경험 지표에 직접적인 이점을 제공합니다.

### 구현 방법

[이벤트 전달](/help/blueprints/use-case-patterns/audience-building-activation/event-forwarding.md) 패턴을 사용하여 Adobe Experience Platform Web SDK에서 수집한 이벤트를 Edge Network을 통해 구성된 서버측 대상으로 라우팅합니다. 이는 클라이언트측의 각 대상에 대해 별도의 태그를 관리하는 것이 아니라 단일 수집 지점에서 서버 간 이벤트 배포를 목표로 하는 경우 올바른 패턴으로, 페이지 가중치를 추가하고 시스템 간에 데이터 불일치를 만듭니다.

### 기술 고려 사항

- Edge Network 이벤트를 전달하려면 이벤트 컬렉션을 Adobe Experience Platform Web SDK 또는 Mobile SDK으로 마이그레이션해야 합니다. 전달 대상을 구성하기 전에 기존 태그 기반 구현의 호환성을 평가해야 합니다.
- 전달 규칙은 각 대상에 필요한 필드만 전송하도록 구성해야 합니다. 작은 필드 하위 집합만 필요한 대상에 전체 XDM 페이로드를 전달하지 마십시오. 이렇게 하면 데이터 전송 비용이 증가하고 규정 준수가 노출됩니다.
- 대상 커넥터는 오류를 정상적으로 처리해야 합니다. 이벤트 전달 파이프라인은 중단 중 데이터 손실을 방지하기 위해 사용할 수 없게 된 대상에 대한 재시도 논리 및 경고를 구현해야 합니다.
- 데이터 거버넌스 정책은 에지에서 캡처한 사용자 동의 환경 설정이 전달 구성에서 적용되는지 확인하기 위해 각 전달 대상에 대해 검토되어야 합니다.
